## Model the challenge as a state machine first

When a login bypass challenge looks small, it is tempting to stare at one parameter too early. I get better results by mapping the flow first:

- what unauthenticated requests return,
- what malformed input returns,
- what happens when the user exists but the password is wrong,
- whether a “failed” branch still changes cookies or redirect targets.

Those details reveal whether a decisive branch has already been touched.

## Failed payloads belong in the writeup

The most helpful replay notes are often the failed requests, not the final winning one.

```http
POST /api/login HTTP/1.1
Content-Type: application/json

{"username":"admin' -- ","password":"x"}
```

This payload failed, but the failure still taught something important: the backend normalized the input length before the interesting comparison, and a whitelist-like branch was reached before the injected text mattered.

In other words, the failure told us we had not hit the decisive condition yet.

## Do not watch the body alone

I try to compare three signal groups in parallel:

1. **response body** changes,
2. **response headers**, especially extra cookies,
3. **follow-up navigation**, such as a different 302 target.

Some challenges look unchanged at the page layer while the server has already started issuing a different state token.

## The real pivot is often short

In this case the winning path was not a “more complex” payload. It was a smaller branch probe plus state replay. The workflow can be compressed into:

```text
1. collect the baseline error with a normal user
2. trigger a different branch with a boundary-length input
3. compare the response headers
4. freeze the parameter that changes state, then minimize everything else
```

Once the pivot is found, the final bypass stops looking magical.

## What the writeup should preserve

The most useful notes are:

- which failed request proved the branch had changed,
- which header hinted that state was already different,
- which minimal tweak killed a false lead.

That kind of evidence transfers well to the next challenge.
