## WAF 是什么？

WAF（Web Application Firewall，Web 应用防火墙）是一种面向 Web 应用的安全防护设备或服务，用于识别和拦截恶意 HTTP/HTTPS 请求。

为了更好理解，我们可以把Web应用比作一家餐厅：

- 传统的网络防火墙像是大门的保安，只检查来客的身份标识，是否是黑名单（端口/IP是否被禁），符合规则就放行，不符合就拒之门外，不管你身上带了什么。
- WAF则是进一步的安检人员，检查来客有没有携带危险物品（拆解 HTTP 报文、检查 Header、Cookie、POST 体里有没有藏着 SQL 注入、XSS 等攻击代码）

# 传统防火墙和WAF的区别：

WAF与传统的防火墙根本区别在于工作的网络层级不同：

| **维度**       | **传统网络防火墙 (FW)**                | **Web 应用防火墙 (WAF)**                                     |
| -------------- | -------------------------------------- | ------------------------------------------------------------ |
| **OSI 网络层** | 网络层/ 传输层                         | 应用层                                                       |
| **关注数据**   | IP 地址、端口、TCP/UDP 协议            | **HTTP/HTTPS 报文内容**（URL、Cookie、Header、POST 提交的数据） |
| **常见防护**   | 防止非授权端口扫描、IP 拦截、Syn Flood | **SQL 注入、XSS、命令执行、跨站请求伪造 (CSRF)**             |
| **HTTPS 处理** | 只能看到加密后的流量                   | 需要解密 HTTPS（SSL 卸载）才能深度检查内容                   |

#  WAF 主要防御哪些攻击？

WAF 的核心任务是阻断针对应用层漏洞的攻击，主要涵盖 常见 Web 攻击漏洞防护及业务安全防护：

- **代码注入类攻击：**
  - **SQL 注入（SQLi）：** 拦截攻击者向数据库输入框注入恶意 SQL 指令。
  - **跨站脚本（XSS）：** 防止攻击者在网页中插入恶意脚本盗取其他用户 Cookie。
  - **命令/代码注入：** 阻止攻击者通过参数执行服务器操作系统命令。
- **应用层 DDoS（CC 攻击/接口刷频）：** 攻击者短时间内向特定接口（如搜索、登录）发送大量真实 HTTP 请求，挤爆服务器 CPU 或数据库，WAF 能通过频率限制和验证码进行拦截。
- **恶意爬虫与 Bot 防护：** 识别非人类行为，拦截数据刮削（如抢票、抓取价格/文章）以及**撞库**（利用泄露密码自动尝试登录）。
- **网页篡改与敏感信息防泄漏：** 过滤服务器返回的响应，防止数据库报错信息、身份证号、银行卡号等敏感数据泄露给外部。

# WAF 是如何识别攻击的？

现代 WAF 绝非单靠简单的“黑名单”，而是采用**混合检测引擎**：

**规则匹配（Rule-Based）：** 基于正则表达式和已知特征库（如 OWASP CRS）。匹配到如 `' OR '1'='1` 等常见攻击模式立即阻断。

**频率与行为分析：** 统计单一 IP/Session 在单位时间内的请求数。对于超过阈值的异常高频请求，触发限流、拦截或弹出 Captcha 验证码。

**语法/语义分析（Semantic Engine）：** 相比正则更高级。WAF 通常结合规则匹配、特征库、归一化、行为分析和部分机器学习能力进行检测。

**机器学习与 AI 引擎：** 建立正常流量的基线模型，通过上下文特征自动化识别未知的 0-day 漏洞攻击和复杂 Bot 行为。

# WAF 的局限性

**提示：** WAF 只是“外挂”式防护，不能完全替代安全编码。

- **无法防御逻辑漏洞：** 例如越权访问（A 用户直接修改 URL 参数查看 B 用户的订单），因为这属于业务合法 HTTP 请求，WAF 很难从语法上判断为非法。
- **存在误报与漏报：** 规则写得太严格可能影响正常用户下单/提交表单（误报）；攻击者使用复杂的编码混淆技术（如多重 URL 编码、字符集切换）有可能绕过 WAF（漏报）。

# 思考： 为什么“频率控制”更适合由 WAF 来做？

传统网络防火墙确实也能做限流，但它和 WAF 的限流有着**本质的区别**（L4 层连接级限流 vs L7 层业务级限流）：

### 1. 传统网络防火墙（L4 层）的限流：

- **只看 IP 和 TCP 连接：** 例如“限制单个 IP 每秒最多建立 100 个 TCP 连接”或“限制 ICMP/SYN 数据包频率”。
- **局限性：** * **无法识别具体业务：** 传统防火墙看不到 HTTP 请求的 URL。如果攻击者只建立 **1 个 TCP 连接**，但在里面并发发送了 10,000 个 HTTP 请求（HTTP 管道化或 HTTP/2 复用），传统防火墙会认为这只是 1 个正常连接，完全拦截不到！
  - **容易误杀：** 如果某个公司/学校几百人用同一个出口公网 IP（NAT 网关）访问网站，传统防火墙按 IP 一锁，会导致整栋楼的正常用户全部无法访问。、

### 2. WAF（L7 应用层）的限流：

- **能识别细粒度的业务路由与身份：**
  - **按接口限流：** 普通页面刷新允许 1 秒 10 次，但“查询数据库/发送短信验证码”等高危接口只允许 1 秒 1 次。
  - **按 Session / Cookie / User-Agent 限流：** 即使几百人共享同一个 IP，WAF 可以根据不同的用户 Cookie 分别计数，精准惩罚恶意账号，而不误伤同 IP 的其他正常用户。登录爆破 / 接口刷频更适合由 WAF 做，因为它能看到 URL、参数、Cookie、Session 等业务上下文。

# 区分：

1. DDOS主要是消耗资源，高频点击搜索，批量刷新。端口爆破（WAF），低计算/高频验证，单次密码验证对 CPU 开销较小，但会产生大量的网络连接和安全日志（防火墙）。
